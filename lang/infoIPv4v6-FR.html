IPv6

Résumé:

  • La plus grande taille allouable possible est /32, mais est réservé pour les providers internet.
  • /48 est une taille typique pour une grand une entreprise. Dans certains cas une taille plus grande peut être fournie mais avec une raison précise.
  • De plus au dessus /48 les IP doivent être declarées au RIR (exemple pour l'Europe au RIPE NCC) avec un plan d'implémentation.
  • Dans IPv6 le nombre d'appareils n'est pas important,' IP étant formée de l'ID de l'interface, donc le subnet standard est /64.
  • L'exception est pour les routeurs (/127). [https://tools.ietf.org/html/rfc6177]
  • /48 a 65536 subnets de /64.
  • ::1/128 est l'adresse loopback
  • ::/128 est l'IP indéfinie, par exemple pas d'allocation possible ou une erreur.
  • LLA = Link-Local-Adresses. fe80::/10. lien entre routeur, switchs.
  • ULA sont fc00::/7 (fc00… - fdff…) utilisé pour des communications internes (comme avant 192.168.x.x). Exemple lien entre compagnies ou tunnel.

    Quelques trucs:

  • https ou http avec IPv6. https://[IPv6]/.... exemple http://[2001:db8:4003:c00::6a] (les IP avec '2001:db8:' sont reservées pour la documentation)
  • https ou http avec IPv6 et port. https://[IPv6]:port/.... exemple http://[2001:db8:4003:c00::6a]:8080

    • Vocabulaires:

  • IANA = responsable pour la coordination globale l'internet.
  • RIR = Registre régional d'internet. RIPE NCC (Europe,Middle East,Central Asia), ARIN (Canada,USA), LACNIC (Latin America), AFRINIC (Africa), APNIC (Asia/Pacific)
  • LIR = Registre local pour internet. En fait le direct provider de votre compagnie.
  • PI = Provider Independent Assignments
  • LLA = Link-Local-Adresses fe80::/10
  • ULA = Unique Local Addresses = fc00::/7 (fc00… - fdff…) utilisé pour des communications internes.
