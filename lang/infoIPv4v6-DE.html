IPv6 DE

Zusammenfassung:

  • Die grösste Mask /32 is reserviert für die Provider.
  • Eine übliche Vergabe ist /48 für einen wichtigen Kunden. Ein grösseres Netzwerk kann mit einem guten Grund vergeben werden.
  • Alle Netzwerke über /48 müssen bei RIR mit einer Imlementierungsplan registriert werden (Zum Beispiel für Europa RIPE NCC).
  • Die Anzahl von Geräten ist nicht wichtig, weil die IP ist die ID-Interface. Deshalb das Standard-Subnet ist /64.
  • Die einzige Ausnahme /127 ist für die Router. [https://tools.ietf.org/html/rfc6177]
  • /48 = 65536 subnets of /64.
  • ::1/128 ist die loopback Adresse
  • ::/128 ist die unbestimmte IP, zum Beispiel wenn die IP nicht vorhanden ist oder fehlerhaft.
  • LLA fe80::/10. Verbindungen zwischen Router, Switchs.
  • ULA fc00::/7 (fc00… - fdff…) benützt für interne Kommunikation (wie vorher 192.168.x.x). Zum Beimspiel Verbindungen zwischen Firmen oder Tunnel.

    Tips:

  • https oder http mit IPv6. https://[IPv6]/.... Beispiel http://[2001:db8:4003:c00::6a] (die IP mit '2001:db8:' sind für Dokumentation reserviert)
  • https oder http mit IPv6 und Port. https://[IPv6]:port/.... Beispiel http://[2001:db8:4003:c00::6a]:8080

    • Einige Ausdrücke:

  • IANA = Verantwortlich für die globale Koordination von der Internet IP Adressierung.
  • RIR = Regional Internet Registrierungszentrum. RIPE NCC (Europe,Middle East,Central Asia), ARIN (Canada,USA), LACNIC (Latin America), AFRINIC (Africa), APNIC (Asia/Pacific)
  • LIR = Lokal Internet Registrierungszentrum. Im End Effekt der Provider Ihres Betriebes. Sie sind der Endkunde.
  • PI = Provider Independent Assignments
  • LLA = Link-Local-Adresses fe80::/10
  • ULA = Unique Local Addresses = fc00::/7 (fc00… - fdff…) benützt für interne Kommunikation.
