Zusammenfassung:

Die grösste Mask /32 is reserviert für die Provider.
Eine übliche Vergabe ist /48 für einen wichtigen Kunden. Ein grösseres Netzwerk kann mit einem guten Grund vergeben werden.
Alle Netzwerke über /48 müssen bei RIR mit einer Imlementierungsplan registriert werden (Zum Beispiel für Europa RIPE NCC).
Die Anzahl von Geräten ist nicht wichtig, weil die IP ist die ID-Interface. Deshalb das Standard-Subnet ist /64.
Die einzige Ausnahme /127 ist für die Router. [https://tools.ietf.org/html/rfc6177]
/48 = 65536 subnets of /64.

Einige Ausdrücke:


IANA = Verantwortlich für die globale Koordination von der Internet IP Adressierung.
RIR = Regional Internet Registrierungszentrum. RIPE NCC (Europe,Middle East,Central Asia), ARIN (Canada,USA), LACNIC (Latin America), AFRINIC (Africa), APNIC (Asia/Pacific)
LIR = Lokal Internet Registrierungszentrum. Im End Effekt der Provider Ihres Betriebes. Sie sind der Endkunde.
PI = Provider Independent Assignments

Detail:

IPv6 assignation IANA->RIR->LIR->customer

Formatierung:

Prefix FF00::/8. Eine IPng multicast Adresse ist Identfikation für eine Gruppe.
Eine interface kann sich an irgendwelche Gruppe anmelden. Hier die Formatierung für die Adressen Multicast.
      |   8    |  4 |  4 |                  112 bits                   |
      +------ -+----+----+---------------------------------------------+
      |11111111|FLGS|SCOP|                  GROUP ID                   |
      +--------+----+----+---------------------------------------------+
      Prefix
       11111111 at the start of the address identifies the address as being a multicast address.
      FLGS
      It is a set of 4 flags: |0|0|0|T| +-+-+-+-+
        The high-order 3 flags are reserved, and must be initialized to 0.
        T=0 indicates a permanently assigned (well-known) multicast address, assigned by the global internet numbering authority.
        T=1 indicates a non-permanently assigned (transient) multicast address.
      SCOP
         SCOP is a 4-bit multicast scope value used to limit the scope of the multicast group. The values are:
      0 Reserved 8 Organization-local scope 1 Node-local scope 9 (unassigned) 2 Link-local scope A (unassigned)
      3 (unassigned) B (unassigned) 4 (unassigned) C (unassigned) 5 Site-local scope D (unassigned) 6 (unassigned) E Global scope
      7 (unassigned) F Reserved.
      GROUP
      GROUP ID identifies the multicast group, either permanent or transient, within the given scope.
