IPv6 EN

RFC 4291

Summary:

  • The biggest allocation size is /32 which is reserved for internet provider.
  • Typically a big customer get /48, but could be bigger when requested with good reasons.
  • Above /48 it needs a registration by RIR (example for Europa RIPE NCC) with implementation plan.
  • IPv6 does not care like IPv4 about the number of devices. The standard subnet is /64 because the interface ID is 64 bits.
  • Exception on routers (/127). [https://tools.ietf.org/html/rfc6177]
  • /48 = 65536 subnets of /64.
  • ::1/128 is the loopback address
  • ::/128 is the undefined IP, for example when cannot be allocated.
  • LLA = Link-Local-Adresses. fe80::/10. link between routers, switchs.
  • ULA fc00::/7 (fc00… - fdff…) uses for internal customer communication (like before 192.168.x.x). Example internal link between to customers or tunnel.

    Tips:

  • https or http with IPv6. https://[IPv6]/.... example http://[2001:db8:4003:c00::6a] (IP beginning with '2001:db8:' are reserved for documentation)
  • https or http with IPv6 and port. https://[IPv6]:port/.... example http://[2001:db8:4003:c00::6a]:8080

    • Some Terms:

  • IANA = responsible for global coordination of the Internet Protocol addressing systems
  • RIR = Regional Internet Registry. RIPE NCC (Europe,Middle East,Central Asia), ARIN (Canada,USA), LACNIC (Latin America), AFRINIC (Africa), APNIC (Asia/Pacific)
  • LIR = Local Internet Registry. In fact the direct provider of your compagny or as end user.
  • PI = Provider Independent Assignments
  • LLA = Link-Local-Adresses fe80::/10
  • ULA = Unique Local Addresses = fc00::/7 (fc00… - fdff…) uses for internal customer communication.
