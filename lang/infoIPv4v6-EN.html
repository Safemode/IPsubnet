IPv6 EN

Summary:

The biggest allocation size is /32 which is reserved for internet provider.
Typically a big customer get /48, but could be bigger when requested with good reasons.
Above /48 it needs a registration by RIR (example for Europa RIPE NCC) with implementation plan.
IPv6 does not care like IPv4 about the number of devices. The standard subnet is /64 because the interface ID is 64 bits.
Exception on routers (/127). [https://tools.ietf.org/html/rfc6177]
/48 = 65536 subnets of /64.

Tips:


Some Terms:


IANA = responsible for global coordination of the Internet Protocol addressing systems
RIR = Regional Internet Registry. RIPE NCC (Europe,Middle East,Central Asia), ARIN (Canada,USA), LACNIC (Latin America), AFRINIC (Africa), APNIC (Asia/Pacific)
LIR = Local Internet Registry. In fact the direct provider of your compagny or as end user.
PI = Provider Independent Assignments

Detail:

IPv6 assignation IANA->RIR->LIR->customer

Format:

Prefix FF00::/8. A IPng multicast address is an identifier for a group of interfaces.
A interface may belong to any number of multicast groups. Multicast addresses have the following format:
      |   8    |  4 |  4 |                  112 bits                   |
      +------ -+----+----+---------------------------------------------+
      |11111111|FLGS|SCOP|                  GROUP ID                   |
      +--------+----+----+---------------------------------------------+
      Prefix
       11111111 at the start of the address identifies the address as being a multicast address.
      FLGS
      It is a set of 4 flags: |0|0|0|T| +-+-+-+-+
        The high-order 3 flags are reserved, and must be initialized to 0.
        T=0 indicates a permanently assigned (well-known) multicast address, assigned by the global internet numbering authority.
        T=1 indicates a non-permanently assigned (transient) multicast address.
      SCOP
         SCOP is a 4-bit multicast scope value used to limit the scope of the multicast group. The values are:
      0 Reserved 8 Organization-local scope 1 Node-local scope 9 (unassigned) 2 Link-local scope A (unassigned)
      3 (unassigned) B (unassigned) 4 (unassigned) C (unassigned) 5 Site-local scope D (unassigned) 6 (unassigned) E Global scope
      7 (unassigned) F Reserved.
      GROUP
      GROUP ID identifies the multicast group, either permanent or transient, within the given scope.
